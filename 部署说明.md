# 黑山石宣传站 — 部署与维护说明

## 一、文件清单

| 文件 | 作用 | 是否要改 |
|---|---|---|
| `index.html` | 网站主页（全部内容都在这一个文件里） | 替换图片/文案时改 |
| `favicon.svg` | 浏览器标签页小图标 | 不用动 |
| `CNAME` | 绑定域名 heishanshi.com（GitHub Pages 必需） | 不用动 |
| `robots.txt` | 告诉搜索引擎可以收录 | 不用动 |
| `sitemap.xml` | 网站地图，提交给百度/Google 加快收录 | 不用动 |

## 二、上线步骤（GitHub Pages）

1. 在 GitHub 新建一个仓库（公开），把以上所有文件传上去（根目录，不要套文件夹）。
2. 仓库 → Settings → Pages → Source 选 `Deploy from a branch`，分支选 `main`、目录选 `/ (root)`，保存。
3. 因为有 `CNAME` 文件，Custom domain 会自动填上 `heishanshi.com`。
4. 等几分钟，勾选 `Enforce HTTPS`。

## 三、Cloudflare DNS 配置（关键，有坑）

域名在 Cloudflare，需要手动加解析指向 GitHub Pages：

**根域名 heishanshi.com → 加 4 条 A 记录：**
```
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```
**www → 加 1 条 CNAME 记录：** 指向 `你的用户名.github.io`

⚠️ 两个常见坑：
- **先把这些记录的橙色云朵点成灰色（DNS only）**，等 GitHub 那边证书签发成功、网站能正常 https 打开后，再决定是否开回橙色代理。
- 若开橙色代理，必须把 Cloudflare → SSL/TLS → 加密模式设为 **Full（完全）**，不能用 Flexible，否则会一直重定向打不开。

## 四、日常维护（最常做的两件事）

### 1. 换图片（最重要）
现在站里所有图都是灰色占位框，写着「待替换：xxx」。
- 把实拍图传到仓库（建议建个 `img/` 文件夹）。
- 在 `index.html` 里找到对应的 `<div class="ph" data-ph="待替换：xxx"></div>`，
  换成 `<img src="img/你的图.jpg" alt="黑山石xxx" loading="lazy">`。
- `alt` 里写上带关键词的描述（如 `alt="黑山石中式园林置石"`），对 SEO 有帮助。
- **矿口/料场实拍、装车实拍最该优先放**——这是建立信任的命门。

## 五、SEO 下一步（按优先级）
